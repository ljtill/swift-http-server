Project Overview

This Swift HTTP Server represents a modern approach to static file serving, combining the performance of SwiftNIO with the safety and expressiveness of Swift 6. Built from the ground up with security, performance, and developer experience in mind.

Design Philosophy

The server embodies several key principles:

  • Security by Default: All inputs are validated and sanitized automatically
  • Performance First: Non-blocking I/O and efficient memory management
  • Developer Friendly: Simple CLI interface with sensible defaults
  • Production Ready: Comprehensive logging and graceful error handling

Built With

  • Swift 6.2+ - Modern Swift with strict concurrency and memory safety
  • SwiftNIO - Apple's high-performance, event-driven networking framework
  • ArgumentParser - Type-safe command-line argument parsing

Architecture

The server follows a clean, modular architecture that separates concerns and enables reusability:

  • HttpServerCore - Core HTTP server functionality as a reusable library
  • HttpServer - Command-line interface wrapper for standalone usage
  • Separation of Concerns: Network layer, file handling, security, and logging are isolated
  • Testability: 9 comprehensive tests across unit, integration, and end-to-end scenarios

Technical Deep Dive

SwiftNIO Integration

The server leverages SwiftNIO's channel pipeline architecture for maximum performance:

  • Event Loop Groups: Optimal thread utilization based on CPU cores
  • Channel Handlers: Custom HTTP file handler for request processing
  • Async/Await: Modern Swift concurrency for clean asynchronous code
  • Back Pressure: Automatic flow control for large file transfers

MIME Type Detection

Essential web content support with automatic content-type detection:

  • Core File Types: HTML, CSS, JavaScript, JSON, and basic images
  • Charset Handling: Proper UTF-8 encoding for text content
  • Extensible: Easy to add new MIME type mappings
  • Fallback: Safe default for unknown file types

Security Features

Security is paramount in any web server. This implementation includes multiple layers of protection against common attack vectors:

Path Security

  • Directory Traversal Protection: Prevents access outside document root using path sanitization
  • Path Normalization: Resolves symbolic links and relative path components safely
  • Input Sanitization: Removes dangerous path elements like .. and .
  • Validation Pipeline: Multi-stage path validation ensures no bypass attempts succeed

Request Handling

  • Method Validation: Only GET requests are accepted, preventing unwanted modifications
  • Header Sanitization: Safe handling of HTTP headers and metadata
  • Error Information: Minimal error disclosure to prevent information leakage
  • Resource Limits: Automatic protection against resource exhaustion

Logging & Monitoring

  • Request Tracking: Complete audit trail of all server interactions
  • Security Events: Detailed logging of security-relevant events
  • Simple Format: Console logs with timestamps for local development
  • Error Details: Clear error messages for debugging
